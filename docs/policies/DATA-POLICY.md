# RDM Digital LTOS — Política Operativa de Datos

## Principios rectores

Prioridad: Seguridad → Estabilidad → Observabilidad → Gobernanza → Escalabilidad → Experiencia → Narrativa. Nada crítico se cambia sin evidencia, rollback y validación mínima.

## Datos personales

- Se recogen sólo los estrictamente necesarios para funcionalidad (auth, contacto, operación de cuentas).
- Nunca se venden, ceden ni explotan para perfiles comerciales externos.
- Está prohibido almacenar datos sensibles sin consentimiento explícito, finalidad documentada, base legal y controles técnicos reforzados.

## Datos sensibles

Se consideran sensibles: salud, biometría, religión, orientación política, información financiera, documentos oficiales, contraseñas, geolocalización precisa persistente, datos de menores y cualquier dato que pueda generar discriminación o daño si se expone.

## Datos estadísticos

- Se recogen métricas agregadas de flujos, visitas, rutas, consultas, comportamiento de red, sensores y eventos de producto.
- Se usan para mejorar servicios, seguridad, resiliencia, recursos municipales y planificación territorial.
- Se aplica anonimización, minimización y prevención de re-identificación.

## IA y analítica

Los sistemas Isabella/Realito y módulos de analítica sólo pueden usar señales agregadas o anonimizadas para mejora operativa, seguridad y calidad del servicio. Cualquier uso individual requiere consentimiento explícito y justificación funcional.

## Validación previa a cambios

Todo cambio relevante sobre datos debe documentar evidencia, beneficio vs riesgo, rollback, validación, módulos afectados, TCI y exposición de telemetría.
